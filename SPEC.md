# 專案規格：一日時間視覺化工具

## 專案概述

### 這個工具在做什麼
一個網頁版的視覺化工具，幫助使用者以「時間容量」的概念規劃一天的任務。使用者透過建立不同大小的時間區塊（block）來表示任務，並將這些區塊放入「一天」的容器中，直觀地感受一天的時間負荷。

**核心概念：**
- 時間視覺化：任務不是列表，而是有「體積」的區塊
- 容量感知：重點是「一天能裝多少」，不是「幾點做什麼」
- 無壓力規劃：不評價、不限制，純粹的規劃輔助工具

### 與傳統待辦清單/行事曆的差異
| 傳統工具 | 本工具 |
|---------|--------|
| 精確時間（9:00-10:00） | 模糊時長（需要 2 小時） |
| 排程導向 | 容量導向 |
| 超時會警告 | 允許超載，讓使用者自己感受 |

---

## 基本使用流程

```
1. 使用者開啟網頁
   ↓
2. 建立任務
   - 輸入任務名稱（例：「寫報告」）
   - 選擇預估時間（例：2 小時）
   - 任務變成一個時間區塊
   ↓
3. 將區塊拖入「今日」畫面
   - 區塊會依照時長顯示不同大小
   - 多個區塊堆疊，視覺上填滿「一天」
   ↓
4. 觀察與調整
   - 一眼看出一天是否「太滿」
   - 可移除區塊或調整時間
   - 繼續新增直到滿意
```

## MVP 最小可行功能

### 必須有的功能

#### 1. 任務建立區
- [ ] 輸入欄位：任務名稱（文字輸入框）
- [ ] 時間選擇：預估時長（下拉選單或按鈕）
  - 建議選項：0.5小時、1小時、1.5小時、2小時、3小時、4小時
- [ ] 「新增」按鈕
- [ ] 建立後的任務顯示為待放置的區塊（任務池）

#### 2. 任務區塊（Block）
- [ ] 視覺化呈現
  - 顯示任務名稱
  - 區塊高度/大小反映時長
  - 不同任務有視覺區隔（邊框或顏色）
- [ ] 互動功能
  - 可拖曳（drag）
  - 可刪除（刪除按鈕或互動）

#### 3. 一日容器
- [ ] 視覺化「一天」的容量
  - 固定高度的容器（代表 24 小時或有效工作時間）
  - 可選：顯示參考線（例：「8小時」、「16小時」）
- [ ] 接收拖入的區塊
  - 區塊進入容器後依序堆疊
  - 超過容器高度時不阻擋，讓視覺上「溢出」

#### 4. 基本互動
- [ ] 拖曳功能
  - 從任務池拖曳到一日容器
  - 從一日容器拖回任務池（移除）
- [ ] 刪除功能
  - 任務池中的區塊可刪除
  - 一日容器中的區塊可移除

## 技術限制與設計原則

### 技術限制
- **前端靜態網站**：可部署到 GitHub Pages
- **無後端**：不需要伺服器
- **資料儲存**：使用 localStorage（關閉瀏覽器後保留）

### 設計原則
1. **簡單直覺**：不需要說明書就能使用
2. **視覺優先**：看得到的比數字重要
3. **無壓力**：不評價使用者的選擇
4. **快速試驗**：輕鬆新增、移除、調整

---

## 不包含的功能（明確排除）

- ❌ AI 建議或自動規劃
- ❌ 使用者帳號系統
- ❌ 提醒或通知功能
- ❌ 多天規劃或行事曆視圖
- ❌ 與他人協作或分享
- ❌ 精確的時刻排程（幾點幾分）
- ❌ 任務優先級或分類系統
- ❌ 統計或數據分析

## 成功標準

1. ✅ 打開網頁，看到清楚的介面
2. ✅ 新增 3 個不同時長的任務
3. ✅ 將任務拖入「今日」容器
4. ✅ 一眼看出今日規劃的時間負荷
5. ✅ 移除或調整任務
6. ✅ 重新整理網頁後，資料還在
